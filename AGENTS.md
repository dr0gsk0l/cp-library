# Repository Guidelines

## このドキュメントについて
貢献者向けガイドをセクションごとに分割し、用途別に参照しやすくしました。以下のリンクから必要なトピックを開いてください。各ページはいずれも 1 分程度で読める分量にまとめています。

## セクション一覧
- [プロジェクト構成](docs/contributor/structure.md): ディレクトリやモジュールの整理方針、`ac-library/` サブモジュールの扱い。
- [ビルドと検証手順](docs/contributor/build-test.md): `oj-verify run/all/docs` などの主要コマンドと実行タイミング。
- [コーディング規約](docs/contributor/coding-style.md): `.clang-format` 設定、命名規則、テンプレート引数の書き方。
- [テスト指針](docs/contributor/testing.md): `.test.cpp` ファイルの配置ルール、テストケース設計、`oj test` の使い方。
- [コミット & PR ルール](docs/contributor/workflow.md): コミットメッセージの書式、PR 説明の必須項目、ドキュメント変更時の注意点。

## 使い方のヒント
- 新しいアルゴリズムを追加するときは、まず構成ガイドとコーディング規約を確認し、命名と配置を決めてから実装を始めるとズレが起きません。
- 実装後はビルド・検証手順を順番にこなし、テスト指針どおりに `.test.cpp` を追加してください。
- 最後にコミット & PR ルールを参照して説明文やログを揃えると、CI とレビューがスムーズになります。***
