# テスト指針

- 各モジュール追加時には最低1本の `.test.cpp` を用意し、対応するコンテストディレクトリに置きます。問題名が衝突する場合は `ABC001A_2.test.cpp` のように末尾へ番号を追加してください。
- 優先する問題ソースは AtCoder、AOJ、library-checker、yukicoder。極端な制約や乱択生成を伴う場合はコメントで手順と想定実行時間を説明します。
- CI の実行制限は 100000ms なので、テストは決定的にし、重いケースは縮小データを併用します。必要に応じて複数ファイルに分割して時間を平準化します。
- ローカルで in/out ファイルを使う場合は `oj test` で確認し、リポジトリには `.test.cpp` のみをコミットします。大きな入力は `generate.cpp` など別プログラムで再生成できるようにします。
- `OJ-VERIFY` の挙動が不安定な問題は `// @ignore` ではなく、原因と回避策を README に記述してから例外的に除外を検討してください。***
