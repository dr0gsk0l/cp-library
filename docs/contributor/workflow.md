# コミット・PR ガイドライン

- コミットメッセージは命令形、先頭大文字、60文字前後を目安にします。影響範囲は `geometry: add half-plane intersection` のようにサフィックスで示します。
- 自動検証コミットは `[auto-verifier] verify commit <sha>` を踏襲し、人手による修正との違いを明確にしてください。
- PR 説明には以下を含めます。1) 変更概要、2) 参照した問題リンクまたは `oj-verify` ログ、3) 必要なシークレット（`YUKICODER_TOKEN`, `GH_PAT` など）が Actions に設定されていることの確認。
- ドキュメント変更時はレンダリング結果のスクリーンショットや GitHub Pages のプレビュー URL を貼り、レビュアーが差分を確認しやすいようにします。
- 複数モジュールに跨る修正は論理的に分割し、小さなコミットにまとめてレビュー負荷を減らしてください。***
